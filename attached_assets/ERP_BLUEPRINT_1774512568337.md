## 1. System Overview

### Architecture
- **Frontend**: Angular SPA (can be replicated with Next.js/React)
- **Backend**: REST API (gateway servers, versioned v4.2)
- **Auth**: JWT-based, 2-step login (email → password)
- **Multi-tenant**: Per-company databases with user-level access
- **Multi-branch**: Branch-level data isolation with HQ consolidation
- **Theming**: 8 color themes (Default, Orange, LightGreen, Dark Green, Purple, Magenta, Brown, Blue)
- **i18n**: 5 languages (English, Hindi, Telugu, Malayalam, Punjabi, Gujarati)
- **Financial Year**: Indian FY (April-March), configurable

### Core Capabilities
- Billing & Invoicing (GST-compliant)
- Inventory Management (multi-store, multi-branch)
- Accounting (double-entry)
- CRM (loyalty, prescriptions for pharmacy)
- Banking Integration
- POS Mode
- Multi-currency support
- Report generation
- Online Store
- ERP-to-ERP data exchange

## 2. Authentication & Authorization

### Login Flow
1. **Step 1**: User enters email/mobile → `POST ValidLoginNew` → returns user info, gateway URL, app URL
2. **Step 2**: Password entry → `POST LoginNew` → returns JWT token, company list
3. **Redirect**: `books.margbooks.com/login?c=<base64_encoded_credentials>`
4. **Session**: JWT stored in localStorage, idle timeout tracking via `ng2Idle`

### Auth Data Model
```
User Registration → User Master → Company Association → Role Assignment → Menu Permissions
```

### Key API Endpoints
- `POST /v4.2/Loginuser/LoginNew` - Authenticate & get JWT
- `GET /v4.2/UserConfiguration/RoleMenuData` - Get role-based menu permissions
- `GET /v4.2/FavorteMenu/List` - User's favorite/pinned menus
- `GET /v4.2/User/List` - List users for company

### Role-Based Access
- **Control Room**: Master settings for permissions per role
- **Granular permissions**: Create, Edit, View, Delete, Print, Export, Cancel per module
- **User types**: Super Admin, Admin, Cashier, Service staff

## 4. Module Details

### 4.1 Item Master (Product Management)
**Route**: `/item-master/list`
**Fields captured from UI**:
- Description / Print Name
- Barcode
- Packing info
- Company (Supplier/Agency)
- Category
- Salt/Composition
- HSN/SAC Code
- Unit of measurement
- Store location
- MRP, Purchase Rate, Sale Rate
- Tax rates (GST)
- Manufacturer
- Branch association
- Substitute items (F7)
- Old rate history (F6)

**Search options**: Description, Print Name, Barcode, Any Where, Top Sale
**Dropdowns**: Branch selector, Store selector, Packing (Y/N), Agency visibility, Salt visibility, HSN visibility

### 4.2 Ledger Master (Accounts)
**Route**: `/partymaster/list`
**87 ledger accounts** including:
- Cash Account, Bank accounts
- GST accounts (CGST/SGST/IGST Payable & Receivable, Input/Output, RCM)
- Sale/Purchase types (Regular, Export, Deemed Export, Counter)
- Forex Gain/Loss
- Freight, Custom Duty
- Credit/Debit ledgers
- General Ledger
**Actions per ledger**: Edit, PDC view, Ledger statement, Outstanding view

### 4.3 Voucher System (Transactions)
**Common voucher types** with numeric IDs:
| ID | Type |
|----|------|
| 51 | Payment |
| 53 | Receipt |
| 55 | Contra |
| 56 | Debit Note |
| 57 | Credit Note |
| 58 | Journal |
| 62 | PDC Payment |
| 63 | PDC Receipt |

**Common UI pattern**: List view → Create/F2 button → Form → Save/F10

### 4.4 Cash Deposits
**Route**: `/cashbook/CashDeposits`
**Table columns**: S.No., Date, Cash Receipt (₹), Expenses (₹), Net Cash (₹)

### 4.5 Cheque Management
**Route**: `/banking/chequebook`
**Actions**: Create/F2, Modify/F3, Delete, Show All/F7, Cancel Cheque, Reuse Cheque

### 4.6 CRM - Loyalty Management
**Route**: `/crm/loyalty/create-loyalty`
**Configuration**:
- Point Apply on: Item Wise / Bill Amount
- Point Calculation: Fraction / Range of Amount
- Point Redeem Rate
- Minimum Amount threshold
- Items: All Items / Selected Items
- OTP Required: Yes/No
- Point Rounding: Auto/No
