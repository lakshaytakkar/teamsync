Order Queue, Source Splitting, and Vendor Order Management
This is Khushal's warehouse team's daily workspace. When a partner places a product order through Phase B of the partner portal, it lands here for processing. The fulfillment team's job is to take that order, figure out which products come from which supplier, place orders with those suppliers, receive the goods at the warehouse, do quality checks, apply MRP stickers, and ship everything to the partner's store.
The Order Queue is the main page — a list of all incoming orders that need processing, sorted by date (oldest first since they should be processed in order). Each order row shows: order number, partner name and city, order date, number of items, total value, current status, and an urgency indicator (how many days since the order was placed — green for under 3 days, yellow for 3-7, red for over 7).
Order statuses flow through these stages: "New" (just placed, not yet looked at) → "Source Split" (we've figured out which items come from where) → "Vendor Orders Placed" (we've ordered from our suppliers) → "Partially Received" (some items have arrived at warehouse) → "Fully Received" (everything's here) → "QC In Progress" → "QC Passed" → "MRP Tagging" (stickers being applied) → "Ready to Ship" → "Shipped" → "Delivered."
When the fulfillment person clicks an order, they see the Order Processing page. This is where the real work happens. The top section shows order details: partner name, city, delivery address, order date, and total value. Below that, the item list — every product in the order with: product name, quantity ordered, source (which supplier it comes from — this is determined by the product's source field: china_haoduobao, china_allen, india_deodap, india_wholesaledock, india_basketo), and a per-item status tracker.
The first action the fulfillment person does is "Split by Source" — the system automatically groups the order items by supplier. So an order with 15 products might split into: 8 items from Deodap, 4 items from Haoduobao, and 3 items from Basketo. Each group becomes a separate "vendor order" — a sub-order that the team places with that specific supplier. For each vendor group, they can: mark it as "Ordered" (they've placed the order with that supplier), enter a reference number (the supplier's order or invoice number), and enter an expected arrival date.
As items arrive at the warehouse, the fulfillment person updates each vendor group: "Received" with the actual quantity received (which might differ from what was ordered — partial deliveries happen). If everything in a vendor group arrived, it moves to "Fully Received." When all vendor groups for an order are fully received, the order automatically advances to "QC In Progress."
The China Batch page is a special view for consolidating China orders. Since China products take 30-45 days, it doesn't make sense to ship each partner's China items separately — we batch them. This page shows all pending China items across all orders, grouped by supplier (Haoduobao vs Allen). The fulfillment person can select items from multiple partner orders and create a "China Batch" — a consolidated order that goes to one supplier. The batch shows: batch number, supplier, total items, total CBM (for freight calculation), estimated shipping cost, and which partner orders are included. When the batch arrives in India, the team receives it here and the system automatically updates each individual partner's order status.
